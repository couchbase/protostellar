Services

service QueryService

QueryService provides N1QL (SQL++) query execution.

rpc Query
Request: couchbase.query.v1.QueryRequest
Response: couchbase.query.v1.QueryResponse

Query executes a N1QL statement and streams back result rows and metadata. Results are delivered incrementally — rows arrive first, followed by a final response containing MetaData (status, metrics, warnings).

message QueryRequest

QueryRequest contains the N1QL statement and all execution options.

Field Type Description
1 bucket_name optional string

The bucket to scope the query to. Optional for cluster-level queries.
2 scope_name optional string

The scope within the bucket. Only meaningful when bucket_name is set.
3 statement string

The N1QL (SQL++) statement to execute.
4 read_only optional bool

When true, the query is read-only and will not modify any data.
5 prepared optional bool

When true, the query engine uses a prepared (cached) form of the statement.
6 tuning_options optional couchbase.query.v1.QueryRequest.TuningOptions
7 client_context_id optional string

An opaque identifier for this request, echoed back in the response for client-side correlation.
8 scan_consistency optional couchbase.query.v1.QueryRequest.ScanConsistency
9 positional_parameters repeated bytes

Positional parameters for parameterized queries ($1, $2, etc.). Each element is a JSON-encoded value.
10 named_parameters map<string, bytes>

Named parameters for parameterized queries ($name). Keys are parameter names, values are JSON-encoded.
11 flex_index optional bool

When true, use a flex index (full-text search index) for the query.
12 preserve_expiry optional bool

When true, preserve document expiry on mutations performed by this query.
13 consistent_with repeated couchbase.kv.v1.MutationToken

Mutation tokens from prior KV operations. Provides at_plus (read-your- own-writes) consistency — the query will wait until these mutations are indexed.
14 profile_mode optional couchbase.query.v1.QueryRequest.ProfileMode
15 durability_level optional couchbase.query.v1.QueryRequest.DurabilityLevel

enum DurabilityLevel

DurabilityLevel for mutations performed by this query.

Name Number Description
DURABILITY_LEVEL_NONE 0

No durability requirements (default).
DURABILITY_LEVEL_MAJORITY 1

Replicate to a majority of data service nodes.
DURABILITY_LEVEL_MAJORITY_AND_PERSIST_TO_ACTIVE 2

Replicate to majority and persist to the active node.
DURABILITY_LEVEL_PERSIST_TO_MAJORITY 3

Persist to a majority of data service nodes.

ScanConsistency controls index consistency guarantees.

Name Number Description
SCAN_CONSISTENCY_NOT_BOUNDED 0

No consistency required — fastest, but may return stale results.
SCAN_CONSISTENCY_REQUEST_PLUS 1

Wait for all prior mutations to be indexed before executing.
