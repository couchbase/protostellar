Services

service KvService

KvService provides key-value document operations against a Couchbase cluster. Operations are organized into reads, mutations, counters, binary operations, sub-document operations, and replica reads.

rpc Get
Request: couchbase.kv.v1.GetRequest
Response: couchbase.kv.v1.GetResponse

Get retrieves a document by key. Supports field-level projection via the project field and optional compressed responses.

rpc GetAndTouch
Request: couchbase.kv.v1.GetAndTouchRequest
Response: couchbase.kv.v1.GetAndTouchResponse

GetAndTouch retrieves a document and simultaneously resets its expiry.

rpc GetAndLock
Request: couchbase.kv.v1.GetAndLockRequest
Response: couchbase.kv.v1.GetAndLockResponse

GetAndLock retrieves a document and acquires a pessimistic lock for the specified duration. The lock must be released via Unlock or will auto-expire after lock_time_secs.

rpc Unlock
Request: couchbase.kv.v1.UnlockRequest
Response: couchbase.kv.v1.UnlockResponse

Unlock releases a pessimistic lock previously acquired by GetAndLock. Requires the CAS value returned by the locking operation.

rpc Touch
Request: couchbase.kv.v1.TouchRequest
Response: couchbase.kv.v1.TouchResponse

Touch resets a document's expiry without retrieving its content.

rpc Exists
Request: couchbase.kv.v1.ExistsRequest
Response: couchbase.kv.v1.ExistsResponse

Exists checks whether a document exists without retrieving its content. Returns a boolean result and the current CAS.

rpc Insert
Request: couchbase.kv.v1.InsertRequest
Response: couchbase.kv.v1.InsertResponse

Insert creates a new document. Fails if the document already exists. Supports durability requirements.

rpc Upsert
Request: couchbase.kv.v1.UpsertRequest
Response: couchbase.kv.v1.UpsertResponse

Upsert creates or replaces a document unconditionally. Supports preserving the existing expiry on update.

rpc Replace
Request: couchbase.kv.v1.ReplaceRequest
Response: couchbase.kv.v1.ReplaceResponse

Replace updates an existing document. Fails if the document does not exist. Supports optimistic locking via optional CAS.

rpc Remove
Request: couchbase.kv.v1.RemoveRequest
Response: couchbase.kv.v1.RemoveResponse

Remove deletes a document. Supports optimistic locking via optional CAS and durability requirements.

rpc Increment
Request: couchbase.kv.v1.IncrementRequest
Response: couchbase.kv.v1.IncrementResponse

Increment atomically increments a counter document by the specified delta. If the document does not exist and an initial value is provided, it is created with that value.

rpc Decrement
Request: couchbase.kv.v1.DecrementRequest
Response: couchbase.kv.v1.DecrementResponse

Decrement atomically decrements a counter document by the specified delta. If the document does not exist and an initial value is provided, it is created with that value.

rpc Append
Request: couchbase.kv.v1.AppendRequest
Response: couchbase.kv.v1.AppendResponse

Append appends raw bytes to the end of an existing document's value.

rpc Prepend
Request: couchbase.kv.v1.PrependRequest
Response: couchbase.kv.v1.PrependResponse

Prepend prepends raw bytes to the beginning of an existing document's value.

rpc LookupIn
Request: couchbase.kv.v1.LookupInRequest
Response: couchbase.kv.v1.LookupInResponse

LookupIn performs one or more sub-document lookup operations (get, exists, count) on paths within a JSON document. Supports xattr and access_deleted flags.

rpc MutateIn
Request: couchbase.kv.v1.MutateInRequest
Response: couchbase.kv.v1.MutateInResponse

MutateIn performs one or more sub-document mutation operations on paths within a JSON document. Supports insert, upsert, replace, remove, array operations, and counters at the sub-document level.

rpc GetAllReplicas
Request: couchbase.kv.v1.GetAllReplicasRequest
Response: couchbase.kv.v1.GetAllReplicasResponse

GetAllReplicas returns the document from the active node and all replica nodes as a server-streamed response. Each response indicates whether it is from a replica via the is_replica flag.

Enums

enum CompressionEnabled

CompressionEnabled controls whether the server may return compressed content.

Name Number Description
COMPRESSION_ENABLED_OPTIONAL 0

The server may return either compressed or uncompressed content.
COMPRESSION_ENABLED_ALWAYS 1

The server should always return compressed content when possible.

enum DurabilityLevel

DurabilityLevel specifies the durability requirements for mutation operations. Controls how many nodes must acknowledge a write before it is considered successful.

Name Number Description
DURABILITY_LEVEL_MAJORITY 0

The mutation must be replicated to a majority of data service nodes.
DURABILITY_LEVEL_MAJORITY_AND_PERSIST_TO_ACTIVE 1

The mutation must be replicated to a majority of nodes and additionally persisted to disk on the active node.
DURABILITY_LEVEL_PERSIST_TO_MAJORITY 2

The mutation must be persisted to disk on a majority of data service nodes.

enum Operation

The type of sub-document lookup to perform.

Name Number Description
OPERATION_GET 0

Retrieve the value at the specified path.
OPERATION_EXISTS 1

Check whether a value exists at the specified path.
OPERATION_COUNT 2

Count the number of elements in an array or keys in an object at the specified path.

enum Operation

The type of sub-document mutation to perform.

Name Number Description
OPERATION_INSERT 0

Insert a value at the specified path (fails if path exists).
OPERATION_UPSERT 1

Insert or replace a value at the specified path.
OPERATION_REPLACE 2

Replace a value at the specified path (fails if path doesn't exist).
OPERATION_REMOVE 3

Remove the value at the specified path.
OPERATION_ARRAY_APPEND 4

Append a value to the end of an array.
OPERATION_ARRAY_PREPEND 5

Prepend a value to the beginning of an array.
OPERATION_ARRAY_INSERT 6

Insert a value at a specific index in an array.
OPERATION_ARRAY_ADD_UNIQUE 7

Add a value to an array only if it doesn't already exist.
OPERATION_COUNTER 8

Increment or decrement a numeric value at the specified path.

enum StoreSemantic

StoreSemantic controls the document-level behavior of the mutation.

Name Number Description
STORE_SEMANTIC_REPLACE 0

The document must already exist (default).
STORE_SEMANTIC_UPSERT 1

Create the document if it doesn't exist, or update if it does.
STORE_SEMANTIC_INSERT 2

Create the document only if it doesn't exist (fail otherwise).
