Services

service BucketAdminService

BucketAdminService provides CRUD operations for Couchbase buckets.

rpc ListBuckets
Request: couchbase.admin.bucket.v1.ListBucketsRequest
Response: couchbase.admin.bucket.v1.ListBucketsResponse

ListBuckets returns all buckets in the cluster with their configurations.

rpc CreateBucket
Request: couchbase.admin.bucket.v1.CreateBucketRequest
Response: couchbase.admin.bucket.v1.CreateBucketResponse

CreateBucket creates a new bucket with the specified configuration.

rpc UpdateBucket
Request: couchbase.admin.bucket.v1.UpdateBucketRequest
Response: couchbase.admin.bucket.v1.UpdateBucketResponse

UpdateBucket modifies the configuration of an existing bucket. Only mutable properties can be changed (e.g., ram_quota, replicas).

rpc DeleteBucket
Request: couchbase.admin.bucket.v1.DeleteBucketRequest
Response: couchbase.admin.bucket.v1.DeleteBucketResponse

DeleteBucket permanently removes a bucket and all its data.

rpc FlushBucket
Request: couchbase.admin.bucket.v1.FlushBucketRequest
Response: couchbase.admin.bucket.v1.FlushBucketResponse

FlushBucket removes all documents from a bucket while preserving its configuration. Requires flush_enabled to be true on the bucket.

Enums

enum BucketType

BucketType specifies the storage type of the bucket.

Name Number Description
BUCKET_TYPE_COUCHBASE 0

Couchbase bucket — persistent, supports replicas and DCP.
BUCKET_TYPE_EPHEMERAL 1

Ephemeral bucket — in-memory only, no persistence.

enum CompressionMode

CompressionMode controls document compression behavior.

Name Number Description
COMPRESSION_MODE_OFF 0

Compression is disabled.
COMPRESSION_MODE_PASSIVE 1

Accept compressed data but do not actively compress.
COMPRESSION_MODE_ACTIVE 2

Actively compress data during compaction.

enum ConflictResolutionType

ConflictResolutionType specifies how conflicts are resolved during cross-datacenter replication (XDCR).

Name Number Description
CONFLICT_RESOLUTION_TYPE_TIMESTAMP 0

Resolve by highest timestamp (last-write-wins).
CONFLICT_RESOLUTION_TYPE_SEQUENCE_NUMBER 1

Resolve by highest sequence number.
CONFLICT_RESOLUTION_TYPE_CUSTOM 2

Use custom conflict resolution logic.

enum EvictionMode

EvictionMode controls how items are ejected when memory is full.

Name Number Description
EVICTION_MODE_FULL 0

Eject everything (metadata + value) from memory.
EVICTION_MODE_NOT_RECENTLY_USED 1

Eject the least recently used items first (ephemeral buckets only).
EVICTION_MODE_VALUE_ONLY 2

Eject only the value, keeping metadata in memory.
EVICTION_MODE_NONE 3

No eviction — reject writes when memory is full (ephemeral buckets only).

enum StorageBackend

StorageBackend specifies the on-disk storage engine.

Name Number Description
STORAGE_BACKEND_COUCHSTORE 0

Couchstore — the traditional storage engine.
STORAGE_BACKEND_MAGMA 1

Magma — high-performance storage engine for large datasets.
