Services

service RoutingService

RoutingService provides cluster topology updates as a server-streamed response. The stream emits a new WatchRoutingResponse whenever the cluster topology changes.

rpc WatchRouting
Request: couchbase.routing.v2.WatchRoutingRequest
Response: couchbase.routing.v2.WatchRoutingResponse

WatchRouting opens a long-lived stream that delivers routing configuration updates. If bucket_name is specified, the response includes vbucket routing for that specific bucket.

Messages

message ServerRouting

ServerRouting describes the number of servers available for a service, split by network locality.

Field Type Description
1 num_local_servers uint32

Number of servers in the local (same-datacenter) group.
2 num_group_servers uint32

Number of servers in the remote (other-datacenter) group.

message VbucketRouting

VbucketRouting describes the distribution of vbuckets across servers. Used for KV data operations to determine which server handles a given key.

Field Type Description
1 num_vbuckets uint32

Total number of vbuckets for this bucket.
2 local_vbuckets repeated uint32

Mapping of vbucket IDs to local server indices.
3 group_vbuckets repeated uint32

Mapping of vbucket IDs to group (remote) server indices.

message WatchRoutingRequest

WatchRoutingRequest specifies the scope of routing information requested.

Field Type Description
1 bucket_name optional string

When specified, include vbucket routing data for this bucket. When omitted, only service-level routing is returned.

message WatchRoutingResponse

WatchRoutingResponse contains the current cluster routing configuration. Each field is optional — only populated when the corresponding service is available in the cluster.

Field Type Description
1 server_routing optional couchbase.routing.v2.ServerRouting

Routing for the KV (data) service.
2 views_routing optional couchbase.routing.v2.ServerRouting

Routing for the Views service.
3 query_routing optional couchbase.routing.v2.ServerRouting

Routing for the Query (N1QL) service.
4 search_routing optional couchbase.routing.v2.ServerRouting

Routing for the Search (FTS) service.
5 analytics_routing optional couchbase.routing.v2.ServerRouting

Routing for the Analytics (CBAS) service.
6 vbucket_data_routing optional couchbase.routing.v2.VbucketRouting

Vbucket distribution for the KV data service. Only present when a bucket_name was specified in the request.
