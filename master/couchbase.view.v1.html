Services

service ViewService

ViewService provides MapReduce view query execution.

rpc ViewQuery
Request: couchbase.view.v1.ViewQueryRequest
Response: couchbase.view.v1.ViewQueryResponse

ViewQuery executes a MapReduce view query and streams back result rows and metadata. The final streamed response contains MetaData.

Messages

message MetaData (Nested in couchbase.view.v1.ViewQueryResponse )

MetaData contains summary information about the view query results.

Field Type Description
1 total_rows uint64

Total number of rows in the view (before skip/limit are applied).
2 debug bytes

Debug information, if requested. JSON-encoded.

message Row (Nested in couchbase.view.v1.ViewQueryResponse )

Row represents a single result from the view query.

Field Type Description
1 id string

The document ID that produced this row.
2 key bytes

The emitted key from the map function. JSON-encoded.
3 value bytes

The emitted value from the map function (or reduce result). JSON-encoded.

message ViewQueryRequest

ViewQueryRequest specifies the view to query and all execution options.

Field Type Description
1 design_document_name string

The name of the design document (without the '_design/' prefix).
2 view_name string

The name of the view within the design document.
3 skip optional uint32

Number of initial rows to skip before returning results.
4 limit optional uint32

Maximum number of rows to return.
5 scan_consistency optional couchbase.view.v1.ViewQueryRequest.ScanConsistency
6 reduce optional bool

When true, execute the view's reduce function on the results.
7 group optional bool

When true, group reduce results by their keys.
8 bucket_name string

The bucket containing the design document.
9 group_level optional uint32

The number of key components to group by (used with group=true).
10 key optional bytes

Fetch only the row with this exact key. JSON-encoded.
11 keys repeated bytes

Fetch only the rows with these exact keys. Each is JSON-encoded.
12 start_key optional bytes

Start of the key range (inclusive). JSON-encoded.
13 end_key optional bytes

End of the key range. JSON-encoded. Inclusiveness controlled by inclusive_end.
14 inclusive_end optional bool

When true, the end_key is inclusive (default: true).
15 start_key_doc_id optional string

Start returning results from this document ID within the start_key.
16 end_key_doc_id optional string

Stop returning results at this document ID within the end_key.
17 on_error optional couchbase.view.v1.ViewQueryRequest.ErrorMode
18 debug optional bool

When true, include debug information in the response metadata.
19 namespace optional couchbase.view.v1.ViewQueryRequest.Namespace
20 order optional couchbase.view.v1.ViewQueryRequest.Order

Enums

enum ErrorMode

ErrorMode controls behavior when errors occur during view execution.

Name Number Description
ERROR_MODE_CONTINUE 0

Continue processing remaining results even if errors occur.
ERROR_MODE_STOP 1

Stop processing immediately on the first error.

enum Namespace

Namespace selects whether to query production or development views.

Name Number Description
NAMESPACE_PRODUCTION 0

Query production design documents.
NAMESPACE_DEVELOPMENT 1

Query development design documents (prefixed with 'dev_').

enum Order

Order controls the sort order of results.

Name Number Description
ORDER_ASCENDING 0

Return results in ascending key order (default).
ORDER_DESCENDING 1

Return results in descending key order.

enum ScanConsistency

ScanConsistency controls index freshness for the view query.

Name Number Description
SCAN_CONSISTENCY_NOT_BOUNDED 0

No consistency required — fastest, may return stale results.
SCAN_CONSISTENCY_REQUEST_PLUS 1

Wait for all prior mutations to be indexed.
SCAN_CONSISTENCY_UPDATE_AFTER 2

Return current results but trigger an index update after the query.
